My role

In January 2024, I joined the personalization team at Loblaw Digital. Our team designed tools for colleagues that produce personalized campaigns for 1M+ customers across Canada.

Problem

  • Personalized campaign production is repetitive – a unique variant of a campaign is created for every target audience

  • For repetitive search tasks, it became time-consuming for colleagues to find the target item in a dropdown “mega-menu” of 50+ menu items

Solution

Why was this an effective search solution for us?

  • A searchable dropdown was a low-effort alternative to typeahead search

  • Task-analysis validated that our colleagues need to filter the mega-menu after typing a keyword to complete their JTBDs, but providing search suggestions was not necessary

Defining searchable dropdown behaviour

I used searchable dropdown best practices, but adapted for internal tools

The height of the dropdown menu is flexible

  • I learned that ideally, colleagues want to see all menu items immediately on focus: ”If I immediately see the [item] I need to find in the menu, I can just click on it there”.

  • If it is not possible to show all menu items, the menu can be set to a max. height

Dropdown items must be sorted alphabetically

The current experience listed segments in alphabetical order, and this was validated by our colleagues to be a good experience – The [items] are listed in alphabetical order, which is helpful in finding what we need right now”“

Dropdown items must be parsable from anywhere

Dropdown items in our internal systems had legacy names with prefixes that were unintuitive. Our colleagues should be able to ignore prefixes upon search.

Search cannot be case-sensitive

Furthermore, dropdown items in our internal systems had legacy names formatted in camelCase. Our colleagues should not have to remember which characters in the target item are upper or lowercase.

Bold substrings of the result that match the user input

This is a best practice that provides the user with visual feedback of their inputs.

Learnings

Reflections from this 2-week sprint

There is more to search than an input field and a dropdown menu

A searchable dropdown is composed of an input field and a dropdown menu, but I also needed to consider how these two nested components speak to one another. What is the height of the dropdown menu? How should we handle case-sensitivity? Solutions were found by thinking about how our colleagues work and validated as feasible by the tech team.

Component design is not isolated from user-centred design

Intuitively, I think about component design as an exercise in design systems thinking, visual design, and interaction design. It’s less intuitive to imagine how user research has a role in component design. However, task analysis was essential to validating the search solution was effective.
